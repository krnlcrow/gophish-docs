Security Awareness — Ethical Phishing

This was a controlled phishing simulation run by the security team. No real compromise occurred. The goal is awareness, not punishment.

Why we run simulations

  • Improve phishing detection
  • Measure awareness
  • Reduce real attack risk

Common phishing signs

  • Urgent or threatening language
  • Unexpected login requests
  • Suspicious links or attachments
  • Sender/domain mismatch

If you interacted

  • Report the email to Security
  • Change your password if entered
  • Enable / confirm MFA
When unsure, always report. False alarms are better than real incidents.
